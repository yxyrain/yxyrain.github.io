存储持续性 作用域和链接性

存储类别如何影响信息在文件间的共享

  • 自动存储持续性
    • 代表：局部变量，无链接性
    • 存储地点：留出一段内存，构造一个栈，使用结束后，将其释放
    • 寿命短，代码块结束便被释放
    • register ： 以前是建议 CPU 寄存器存储变量，现在加不加一样
  • 静态存储持续性
    • 代表：全局变量 z ， static 变量 q ， 区间 static 变量 k
    • 特点：默认零初始化， 寿命长，一直持续到进程终结
    • 储存地点：固定的内存块，不使用栈进行操作
    • 静态的持续性，外部的链接性：
      • C++ 的 单定义规则(ODR)：变量只能有一次定义机会
      • 两种定义声明
        1. 定义声明：int a;
        2. 引用声明：extern int a;
      • 使用方法：多文件使用外部变量时，用关键字 extern 声明
    • C++的作用域解析运算符 $::$ ， 放在变量名前代表全局版本的变量
    • static 限定符:当用于作用域为全文件的变量时，会使该变量的链接性变为内部链接性
      • 用法：在两个文件中创建名称相同的变量显然是不合乎标准的，这时，使用 static 会使另一个文件的常规外部变量隐藏。

说明符和限定符

  • cv-限定符
    1. const
      • 表明内存被初始化后，程序无法对其进行更改
      • 在 C++ 中， const 全局变量的链接性是内部的
      • const 的内部链接性可以用 extern 进行覆盖
    2. volatile
      • 表明即使程序代码没有对内存进行修改，值依然有可能变化，例如指针指向硬件位置，其中包含了来自串行端口的时间信息，可以避免程序将这个值缓存到寄存器中，从而出现两次同样的值
    3. mutable
      • 可以用 mutable 指出，即使结构体/类变量为 const ，其中的某个成员依然可以被修改

        函数和链接性

  • 所有函数的储存链接性都自动设为静态，且链接性默认为外部链接
  • 可以使用 extern 来说明函数是另外一个文件定义的
  • 可以使用 static 将函数的链接性设置成内部
  • 内联函数不受单定义规则的约束

      类和动态内存分配

介绍如何对类使用 new 和 delete 以及如何处理动态内存带来的问题

  • 不能在类声明中初始化静态成员变量
  • 但如果静态成员成员是 const 整数类型或枚举型，则可以在类中初始化。
  • 类对象作为函数参数传递时，将会产生对象的副本，而这个副本是调用类中的拷贝构造函数实现的，如果类中没有自定义拷贝构造，编译器会自动生成一个缺省的函数，该函数有以下特点：
    • 对于基本类型的成员变量，按字节复制
    • 对于类类型的成员变量(成员子对象)，将自动调用相应类的拷贝构造函数来初始化
    • 指针只是简单的赋值，而不会自动分配内存。

当退出该函数时,副本将调用析构函数，删除该指针,指针所指向内存也已被释放。将原始对象只保留了一个指针值。当在类中释放对象再次调用析构函数，会出现double free 的错误。

两种解决方式：

  1. 自定义拷贝构造函数。
  2. 将函数对象作为引用传递，引用时操作不会释放对象。
